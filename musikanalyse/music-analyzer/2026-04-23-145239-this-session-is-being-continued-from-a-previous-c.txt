⏺ The app is running at http://localhost:5174. Here's a summary of what changed
  and why:

  ---
  Worker (verovio.worker.ts)
  - Now receives chords: BeatChord[] alongside xml in the render message
  - After the first loadData(xml), calls getMEI({ noLayout: 1 }) to get the
  score as MEI without pre-computed layout
  - injectHarmLabels() uses DOMParser + createElementNS(MEI_NS, 'harm') — proper
   namespace, which was the root cause of the previous font-size="0px" bug from
  string-injecting namespace-less elements
  - Each <harm> gets @tstamp (converted from 0-based quarter-note beats using
  the MEI meterSig@unit), @staff (bottom staff, so labels appear below the grand
   staff), and @place="below"
  - Re-renders from the modified MEI so Verovio positions the harm elements at
  the exact notated beat

  ScoreViewer (ScoreViewer.tsx)
  - After the SVGs are in the DOM, queries g.harm elements
  - If Verovio rendered them with a visible font-size → clears the overlay (no
  duplicates)
  - If font-size is still 0 (Verovio quirk) → reads each harm element's position
   via getCTM() converted to CSS pixels — reliable even for invisible elements,
  works across multiple SVG pages
  - Falls back to the old linear-interpolation approach if no harm elements are
  found at all
